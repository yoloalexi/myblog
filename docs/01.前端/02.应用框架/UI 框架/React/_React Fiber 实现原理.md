Fiber 架构想解决的是 React 的虚拟 DOM 树在构建和 DIFF 阶段无法打断的问题。

具体的实现方式有：
1. 双缓冲机制：保留上一次状态和处理中的 Fiber 树。难点在于中断和恢复机制。
2. 两阶段提交：虚拟 DOM 树构建可打断，提交变更不可打断
3. 闲时调度：每一个 Fiber 节点都是一个最小执行单元
