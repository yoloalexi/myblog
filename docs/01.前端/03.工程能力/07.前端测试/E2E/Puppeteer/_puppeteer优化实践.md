# 优化Chromium执行流程

请求到达->启动Chromium->打开tab页->运行代码->关闭tab页->关闭Chromium->返回数据

=>

请求到达->连接Chromium->打开tab页->运行代码->关闭tab页->返回数据

官方并不建议这样做，因为一个tab页阻塞或者内存泄露会导致整个浏览器阻塞并Crash。万全的解决办法是定期重启程序，可参考php-fpm的做法，当请求1000次或者内存超过限制后重启对应的进程。
